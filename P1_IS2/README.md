# Proyecto de Ingeniería de Software II
Este repositorio contiene el desarrollo y la documentación técnica del proyecto académico para la cátedra de Ingeniería de Software II.

## 1. Propósito del Repositorio
El objetivo principal es aplicar metodologías de desarrollo de software, estándares de calidad y control de versiones para la construcción de una solución robusta.

### 1.1 Objetivos Específicos
* Implementar patrones de diseño de software.
* Asegurar la trazabilidad de los requisitos.
* Gestionar el ciclo de vida del desarrollo mediante Git.

### 1.2 Requisitos del Entorno
* Motor de base de datos configurado.
* Entorno de ejecución compatible.
* Conexión a internet para dependencias externas.

## 2. Planificación y Estructura
A continuación se detallan las fases de trabajo y los componentes del sistema.

### 2.1 Fases del Proyecto (Lista Numerada)
1. Análisis de requisitos y definición del alcance.
2. Diseño de arquitectura y diagramas UML.
3. Desarrollo del núcleo de la aplicación.
4. Pruebas unitarias y de integración.

### 2.2 Componentes del Sistema (Lista Ordenada)
1. Interfaz de Usuario (Frontend).
2. Lógica de Negocio (Backend).
3. Capa de Persistencia (Base de Datos).
